# Журнал решений --- Запись 1

**Решение:** выбрана отрасль коммерческой недвижимости (PropTech) и
гипотеза AI-платформы для интеллектуального управления зданием.

**Основание:** в ходе обсуждения пришли к выводу, что такой подход
создает понятную ценность для бизнеса (снижение затрат, аналитика
эксплуатации, прогнозирование поломок) и выглядит сильнее, чем
первоначальная идея повышения продуктивности сотрудников.

# Журнал решений --- Запись 2

**Гипотеза для проверки на людях:** основная ценность продукта ---
помощь руководителю эксплуатации в определении приоритетов среди
большого количества событий инженерных систем.

**Что вскрыл скептик:** первоначальная концепция была слишком широкой и
недостаточно отличалась от существующих решений.

**Решение:** сузить гипотезу и сфокусировать MVP на AI Decision Engine
--- анализе, объяснении и приоритизации эксплуатационных проблем.

# Журнал решений — Запись 3

Решение: **уточняем гипотезу**.

MVP сфокусирован только на приоритизации эксплуатационных проблем и поддержке принятия решений. Гипотеза требует проверки на реальных пользователях.

# Журнал решений — Запись 4

**Решение:** создан HTML-каркас приложения (index.html) — статический прототип без бэкенда.

**Основание:** нужен был быстрый способ проверить ядровой сценарий и показать продукт первым пользователям без разработки полного стека. Чистый HTML/CSS/JS позволяет открыть в браузере без сборки.

**Что реализовано:** 5 экранов (авторизация, дашборд, карточка проблемы, рекомендация, история), навигация между ними, данные по трём демо-проблемам (HVAC №17, Вентиляция В, Датчик CO₂).

**Следующий шаг:** подключение реального бэкенда на Go.

# Журнал решений — Запись 5

**Решение:** зафиксирован технический стек и архитектура MVP.

**Backend:** Go — производительность и типобезопасность для API и интеграционного слоя.
**AI/ML:** Python + FastAPI — богатая ML-экосистема, отдельный сервис.
**БД:** PostgreSQL + TimescaleDB — реляционные данные + time-series для телеметрии инженерных систем.
**Frontend MVP:** HTML+CSS+JS, после подтверждения гипотезы — React+TypeScript.

**Открытые вопросы зафиксированы** в Технической спецификации: выбор первых клиентов, минимальный набор интеграций, место хостинга, ответственный за DevOps.
