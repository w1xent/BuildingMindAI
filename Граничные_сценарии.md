# Граничные сценарии и защита — BuildingMind AI (прототип)

> Анализ альтернативных веток поведения, уязвимостей и реализованной защиты.
> Прототип браузерный (`index.html` + `analysis.js` + `data.json`, состояние в `localStorage`).

---

## Карта смысловых ветвлений

```
Запуск
 ├─ данные загрузились + анализ дал проблемы ──► есть сессия? ──┬─ да ─► Дашборд
 │                                                              └─ нет ─► Вход
 └─ сбой загрузки / file:// / пустые данные ───────────────────► Экран ошибки ► «Повторить»

Вход
 ├─ пустые поля ──────────► «Введите email и пароль»
 ├─ неверные данные ──────► «Неверный email или пароль»
 └─ верно (регистр email не важен) ► сохранить сессию ► Дашборд

Список проблем
 ├─ клик по активной ─────► Карточка
 ├─ клик по «в работе» ───► игнор (карточка некликабельна)
 └─ фильтр без совпадений ► пустое состояние

Карточка → Рекомендация → «Принято в работу»
 ├─ проблема существует ──► статус in_progress, тост, уход из активных
 └─ currentId невалиден ──► редирект на Дашборд
```

## Альтернативные сценарии и защита

| # | Ветка / нестандартное действие | Было | Стало (защита) |
|---|--------------------------------|------|----------------|
| 1 | `data.json` не загрузился (сеть, 404, битый JSON) | Молчаливо пустой дашборд, логин необъяснимо не работал | **Экран ошибки** с кнопкой «Повторить» |
| 2 | Открытие файла двойным кликом (`file://`) | `fetch` падал, белый экран | Экран ошибки с подсказкой «запустите через локальный сервер» |
| 3 | `analysis.js` не подключился | `ReferenceError`, приложение мертво | Перехват → экран ошибки |
| 4 | Данные битой структуры (нет `users`/`systems`, пусто) | Пустой UI, ложное «всё в норме» | Валидация формы → экран ошибки |
| 5 | `localStorage` недоступен (приватный режим, квота) | Необработанное исключение рвало поток (логин/сохранение) | Всё хранилище в `try/catch`, работа «в памяти» |
| 6 | Повреждённый `bm_statuses` (массив/примитив/мусор) | Возможна порча состояния | Проверка типа → откат к `{}` |
| 7 | Повреждённый `bm_user` (нет `id`) | Мог автологинить «пустого» пользователя | Проверка формы сессии → на экран входа |
| 8 | `currentId` устарел/невалиден, переход на карточку | Показ старого/пустого контента | Редирект на Дашборд |
| 9 | `openProblem()` с несуществующим id | Открывался пустой экран | Игнор + предупреждение в консоль |
| 10 | Открытие уже закрытой проблемы | Можно было снова «принять в работу» | Игнор (только активные открываются) |
| 11 | Неизвестный экран в `navigate()` | `TypeError` на `null.classList` | Проверка существования экрана |
| 12 | **XSS** в именах/описаниях/шагах (актуально при реальных данных) | Прямой `innerHTML` без экранирования | `escapeHTML()` на все данные + делегирование вместо inline-`onclick` |
| 13 | Некорректный `priority` из данных | Попадал в имя CSS-класса | `normPriority()` сводит к `high/medium/low` |
| 14 | Нет блока `analysis`/телеметрии у проблемы | Падение на `series[0]` | Блок «как обнаружено» прячется, без падения |
| 15 | Пустые `steps` у рекомендации | Пустой список | Заглушка «Рекомендации не заданы» |
| 16 | Email в другом регистре | Логин не проходил | Сравнение без учёта регистра |
| 17 | Две вкладки, статусы разошлись | Рассинхрон до перезагрузки | Слушатель `storage` — авто-обновление списков |

## Осознанно принятый риск (вне MVP)

Эти ветки **намеренно не закрыты** — они требуют бэкенда и не влияют на проверку гипотезы:

- **Аутентификация ненастоящая.** Пароли в открытом виде в `data.json`, проверка на клиенте. Это прототип — реальная авторизация будет на Go-бэкенде (JWT, хеши).
- **Нет защиты от подделки состояния.** Любой может изменить `localStorage` / `data.json`. Для демо это не угроза.
- **Нет серверной валидации.** Источник данных пока доверенный (статичный файл).
- **Многопользовательность не разделена** — один браузер = одно состояние.

> Эти пункты закрываются на этапе перехода с прототипа на архитектуру из
> [Технической спецификации](Техническая_спецификация.md) (Go + Python + БД).

## Как перепроверить защиту

В консоли браузера (на дашборде):

```js
navigate('detail');                         // currentId не задан → останется dashboard
openProblem('нет_такого');                  // игнор + warn
localStorage.setItem('bm_statuses','[1,2]'); loadStatuses();  // → {}
problemCardHTML({id:'x', system_name:'<img onerror=alert(1)>', // → экранировано
  description:'', priority:'high', status:'active', detectedAt:new Date()});
```
