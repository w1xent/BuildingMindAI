## Что мы строим

**BuildingMind AI** — B2B SaaS платформа для коммерческой недвижимости (PropTech / Smart Building).

**Проверяемая гипотеза:** AI-платформа анализирует данные инженерных систем, выявляет критичные проблемы и помогает руководителю эксплуатации определить приоритет действий.

## MVP — AI Decision Engine

Узкая точка входа: помочь руководителю эксплуатации быстрее понять, **что наиболее критично и что делать в первую очередь**.

### Что делает MVP
1. Получает данные от инженерных систем здания
2. Находит аномалии
3. Приоритизирует проблемы
4. Показывает объяснение и рекомендацию

### Текущая реализация (браузерный прототип)
- `index.html` — UI (5 экранов), сессия и статусы в `localStorage`
- `analysis.js` — движок анализа аномалий: генератор телеметрии → детектор (baseline + z-score / дисперсия / пропажа данных) → приоритизация (`severity × длительность × критичность`)
- `data.json` — конфиг систем (норма, шум, критичность, тип аномалии, интерпретация)
- Список проблем — **результат анализа**, а не захардкоженные данные. На карточке видно «как обнаружено» + спарклайн ряда
- Датчиков пока нет → телеметрия синтетическая (детерминированная, seed от id системы)

### Что НЕ входит в MVP (стоп-лист)
- Управление зданием (автоматическое)
- Digital Twin
- Биометрия
- Управление помещениями
- AI-чат
- Мобильное приложение
- Избыточные интеграции

**Принцип:** если фича не помогает руководителю эксплуатации приоритизировать проблемы — она не в MVP.

## Ядровой сценарий

```
Пользователь открывает систему
→ видит список критичных проблем
→ открывает одну проблему
→ получает объяснение причины и рекомендацию
→ принимает решение о действии
```

## Целевые персоны

### Персона 1 — Руководитель службы эксплуатации
- Контролирует инженерные системы, планирует обслуживание
- **Боль:** большое количество сигналов, ручная расстановка приоритетов

### Персона 2 — Главный инженер объекта
- Обеспечивает бесперебойную работу, реагирует на аварии
- **Боль:** поиск причин неисправностей по данным из разных источников

## Product Vision

Система поддержки принятия решений для эксплуатации коммерческой недвижимости. Начинаем с анализа и приоритизации наиболее важных проблем инженерных систем. Постепенно расширяем возможности, сохраняя главную задачу — помогать принимать обоснованные решения быстрее и с меньшими затратами.

## Стоп-условие гипотезы

Если реальные пользователи не подтвердят ценность приоритизации проблем как основной причины использования продукта — гипотеза требует пересмотра.

## Принципы разработки

- Каждое решение проверяем через ядровой сценарий
- Не расширяем функционал до подтверждения гипотезы
- Backend на Go, AI/ML логика на Python
- Данные инженерных систем — источник истины, не создаём данные без интеграций
- Минимальный набор интеграций для MVP (уточнить)
