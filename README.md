# BuildingMind AI

> **AI Decision Engine для эксплуатации коммерческой недвижимости.**
> Анализирует данные инженерных систем здания, выявляет аномалии,
> расставляет приоритеты и подсказывает, что делать в первую очередь.

B2B SaaS для PropTech / Smart Building. Сейчас в репозитории —
**рабочий браузерный прототип** для проверки продуктовой гипотезы.

---

## 🚀 Быстрый старт

Прототип запускается в браузере без сборки. Нужен только локальный сервер
(из-за загрузки `data.json` через `fetch` — открытие файла напрямую не подойдёт):

```bash
npx serve . -p 3000
```

Откройте `http://localhost:3000`.
Демо-доступ заполнен автоматически: `ivanov@bc-nevsky.ru` / `demo123`.

---

## 🎯 Что делает MVP

1. **Получает данные** инженерных систем (пока — синтетическая телеметрия)
2. **Находит аномалии** — baseline + z-score / дисперсия / пропажа данных
3. **Приоритизирует** проблемы — `severity × длительность × критичность`
4. **Объясняет и рекомендует** — почему критично и что делать

Ядровой сценарий: открыл → увидел список критичных проблем → открыл одну →
понял причину → получил рекомендацию → принял в работу.

## 🗂 Структура репозитория

```
.
├── index.html          # UI прототипа (5 экранов), состояние в localStorage

├── data.json           # Конфиг систем (норма, шум, критичность, аномалия)
├── CLAUDE.md           # Краткая сводка проекта для разработчиков
└── docs/
    ├── Бриф_продукта.md            # Персоны, ценность, метрики, конкуренты
    ├── Техническая_спецификация.md # Архитектура, стек, API, дизайн-система
    ├── Журнал_решений.md           # Хронология ключевых решений
    ├── Сценарий_демо.md            # Гайд для юзабилити-теста
    ├── Граничные_сценарии.md       # Карта ветвлений и защита
    └── архив/                      # Ранние черновики (ванпейджеры, правила v1–v3)
```
